An Introduction to Functions, Execution Context and the Call Stack

Watch video

What I understood from this video is that the way a computer or browser reads and executes code in one line at a time. Because along the code we’ll find also some pieces of code inside of functions the way the computer runs our code follows a pattern.

Going one line at a time is called the Thread of Execution. When the code starts to run it begins in the global execution thread (basically where all of our code is written). Then the global execution is stacked in the Call Stack, that is the place where our computer is going to organize the work that it is doing. So, what happens when the computer reads a line with a variable and a value assigned to it? The browser stores it in the Global Memory for later use. Something similar happens when you declare a function, the computer won’t run code inside of it, instead it stores the function name (or tag) and the code related in the global memory.

Now, if our computer continues reading the code and now encounters a function invocation, the code inside the function will be added to the Call Stack and the computer will execute it. So, a new execution context will start, and our thread of execution is going “inside” the function and starts reading its code one line at a time. This new execution context has its own memory, called local memory, where it will store any variables or even functions it will use. Once the job is done inside the code of the function, the thread of execution goes outside the function back to the global execution context, and the thread of execution continues its way.

How to Understand Callbacks and Higher Order Functions

Watch video

Even though this video was bigger, I think the message was simpler. Higher order functions are functions the accept other functions as arguments or return other functions. Although they have these characteristics, they are just functions. The name Higher Order is just to identify them.

Now a Callback Function is a function that can be passed to another function (to a higher order function).

So, why is it so important to know this? Because we can create more general functions with these concepts in mind. Instead of creating many functions with a single task, we can create more flexible functions that we can change and adapt to our needs (This was taught in the book JS: Novice to Ninja).

There’s an example in the video that demonstrates this. Three functions are created in the video, one to multiply items in an array by two and then returning and array with the numbers multiplied. The second function divides items in an array by two and then returns and array with the numbers divided. The third function adds three to each item in an array and then returns and array with the numbers plus three.

So, what’s the problem? We are defining many functions with similar functionality. We could solve it using what we know about higher order functions and callbacks. We create a function that accepts the array as an argument and a callback. The callback will be called inside the higher order function. The callback function passed is the piece of code that will perform different operations to the array, and the higher order function will return the new array. This way we can use the same function for many different operations; operations that we can define in a callback function.
